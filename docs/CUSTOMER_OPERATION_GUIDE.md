# Keyun Studio Customer Operation Guide

이 문서는 디자인 편집 기능 없이, 제작자가 사이트를 세팅한 뒤 고객이 관리자에서 운영하는 MVP 기준 안내입니다.

## 1. 처음 로그인 후 확인할 것

1. `/dashboard`에서 대표 사이트 상태를 확인합니다.
2. `오픈 전 확인하기`를 눌러 사이트맵과 페이지 구성을 확인합니다.
3. 공개 전에는 `초안 미리보기`로 각 페이지를 확인합니다.
4. 문제가 없으면 사이트 상세에서 `게시하기`를 실행합니다.

## 2. 페이지 관리

- 위치: `/dashboard/sites/{siteId}/sitemap`
- 할 수 있는 일:
  - 1차, 2차, 3차 페이지 추가
  - 페이지 제목, 요약, 본문 수정
  - 메뉴 노출/숨김 설정
  - 페이지 순서 변경
  - 초안 미리보기

운영 팁:
- 메인 페이지는 기본 페이지라 삭제하거나 숨기지 않습니다.
- 첫 사이트 생성 시 `홈`, `서비스` 페이지가 기본 생성됩니다.
- 사이트 생성 단계에서 `포트폴리오`, `오시는 길`을 선택하면 해당 페이지도 함께 생성됩니다.
- 메뉴코드는 공개 주소에 사용되므로 영문 소문자 또는 짧은 한글 주소를 권장합니다.
- 페이지를 수정한 뒤 공개 사이트에 반영하려면 `게시하기`가 필요합니다.

## 3. 게시글 관리

- 위치: `/dashboard/content/posts`
- 할 수 있는 일:
  - 공지사항, 블로그, FAQ 게시글 작성
  - 임시저장, 게시, 예약 상태 관리
  - 게시글 수정/삭제

운영 팁:
- 공개 상태인 게시글만 공개 사이트에 노출됩니다.
- 게시 후 공개 사이트의 게시글 목록과 상세 페이지를 확인합니다.

## 4. 문의 관리

- 위치: `/dashboard/content/forms`
- 할 수 있는 일:
  - 신규 문의 확인
  - 처리 상태 변경: 신규, 처리 중, 완료
  - 내부 메모 저장
  - CSV 내보내기
  - 이메일/전화 바로 연결

운영 팁:
- 신규 문의가 있으면 대시보드와 문의 화면 상단에 알림처럼 표시됩니다.
- 운영자가 이메일 알림을 설정해 둔 경우, 공개 사이트에서 문의가 접수될 때 지정된 메일로도 알림이 발송됩니다.
- 확인한 문의는 `처리 중`으로 바꾸고, 상담이 끝나면 `완료`로 변경합니다.
- 중요한 처리 내용은 내부 메모에 남겨둡니다.

## 5. 미디어 관리

- 위치: `/dashboard/content/media`
- 할 수 있는 일:
  - 이미지, 영상, PDF 업로드
  - 공개 URL 복사
  - 미디어 삭제

운영 팁:
- 너무 큰 파일은 업로드 전 압축합니다.
- 같은 이미지를 여러 번 올리지 않도록 파일명을 정리합니다.

## 6. 팝업 관리

- 위치: `/dashboard/content/popups`
- 할 수 있는 일:
  - 팝업 생성
  - 노출 상태 변경
  - 노출 기간 설정
  - 버튼 링크 설정

운영 팁:
- 이벤트 종료 후 팝업 상태를 비활성화합니다.
- 공개 사이트에서 팝업이 정상 노출되는지 확인합니다.

## 7. 오픈 전 최종 확인

- [ ] 고객 계정으로 로그인됩니다.
- [ ] 디자인 메뉴가 일반 고객에게 보이지 않습니다.
- [ ] 사이트맵의 모든 공개 페이지가 열립니다.
- [ ] 모바일에서 메인/서브/게시글/문의폼이 깨지지 않습니다.
- [ ] 문의폼 접수 후 관리자에 문의가 들어옵니다.
- [ ] 이메일 알림을 사용하는 경우, 테스트 문의 후 알림 메일이 도착합니다.
- [ ] 게시글 공개/비공개 상태가 정상입니다.
- [ ] 팝업 노출 상태와 기간이 정상입니다.
- [ ] SEO 제목, 설명, 대표 이미지가 설정되어 있습니다.
- [ ] 게시 후 `/s/{siteSlug}` 공개 주소가 열립니다.
